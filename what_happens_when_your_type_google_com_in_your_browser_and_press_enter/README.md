# What happens when you type `https://www.google.com` and press Enter?

## Project

This project explores the full infrastructure journey of a single web request, from the moment you press `Enter` to the rendered page appearing in your browser.

## Files

| File | Description |
|------|-------------|
| `0-blog_post.md` | Full blog post covering all 8 infrastructure layers |
| `1-what_happen_when_diagram` | Link to the request flow diagram |

## Concepts covered

- **DNS resolution**: How a domain name is translated into an IP address through recursive lookups
- **TCP/IP**: The 3-way handshake that establishes a reliable connection
- **Firewall**: How packet filtering and DDoS protection work at the network edge
- **HTTPS/SSL**: TLS handshake, certificate validation, and symmetric encryption
- **Load balancer**: How traffic is distributed across backend servers
- **Web server**: How HTTP requests are parsed and routed
- **Application server**: Where business logic runs and HTML is generated
- **Database**: How persistent data is fetched and cached
